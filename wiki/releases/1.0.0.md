# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 is a production-ready prompt interface featuring a dual LLM backend (llama.cpp and Claude API), a modular plugin system, and a 20-round tool loop for complex workflows. This release introduces enterprise-grade integrations for E*TRADE brokerage data, including real-time option chains and portfolio analysis, alongside LiteAPI for semantic hotel search and booking. The architecture supports a two-tier Wiki RAG system for efficient documentation retrieval without embeddings, while development tools enable local code execution, shell commands, and interactive visualizations. Built on Node.js v20+, Express 5.2.1, and Tailwind CSS 4.2.1, this milestone release ensures backward compatibility with v0.1.0 sessions while introducing no database migrations.

## Key topics
- Dual LLM Backend — Seamless switching between local llama.cpp and remote Claude API
- E*TRADE Integration — Full brokerage access with Greeks, cost basis, and transaction history
- LiteAPI Travel — Semantic hotel search, booking workflows, and weather forecasts
- Two-Tier Wiki RAG — Compressed documentation index for efficient retrieval
- Plugin System — Modular tool groups with lazy loading for extensibility
- Development Tools — Source code management, Python execution, and applet visualizations
- Technical Stack — Node.js v20+, Express 5.2.1, Tailwind CSS 4.2.1, Puppeteer
- Known Limitations — Rate limits in sandbox, 25-symbol option chain cap, manual llama.cpp setup
