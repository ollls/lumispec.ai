# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 delivers a production-ready AI assistant featuring a dual LLM backend (local llama.cpp and cloud Claude), a comprehensive suite of self-aware source code tools, and a modular plugin architecture. The release introduces a distinct separation between the TaskMaster prompt decomposer and the Task Pipeline executor, enabling complex multi-step workflows with chained or isolated context handling. Key integrations include full E*TRADE brokerage access for portfolio management and options trading, a robust web research system with three search engines and three fetch modes, and a hierarchical RAG wiki system for self-improving documentation. The system enforces strict safety tiers for code modifications and supports a "Karpathy-style" feedback loop where the agent researches, edits source docs, and rebuilds the wiki index autonomously.

## Key topics
- Source tools — Eight core commands for reading, editing, and versioning code with safety gates
- Task pipeline — Distinct decomposer and executor with chained top-level and isolated subtask contexts
- Web research — Tavily, Keiro, and DuckDuckGo engines with stealth and browser fetch modes
- E*TRADE integration — Full account, transaction, and options trading access with CSV/JSON export
- Wiki RAG — Two-tier system using compressed YAML frontmatter and source markdown for self-improvement
- Dual LLM backend — Configurable routing between local privacy-focused models and cloud APIs
- Plugin architecture — Hot-reloadable modules with dependency graph validation and cycle detection
- Precision mode — Vectorized Python execution rules for data analysis and computation tasks
