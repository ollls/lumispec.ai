# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 introduces a production-ready AI assistant featuring a dual LLM backend supporting both local `llama.cpp` inference and cloud `Claude API` routing. The release highlights the TaskMaster system, which enforces strict context isolation to maintain constant memory footprints for local models during multi-step pipelines. Comprehensive integrations include a three-engine web research system with stealth and browser fetch modes, full E*TRADE brokerage access for options and portfolio management, and LiteAPI travel booking capabilities. Additional features cover a modular plugin architecture with hot-reload support, a hierarchical RAG wiki system, and an inline visualization engine for SVG, Chart.js, and HTML applets.

## Key topics
- Web Research — Tavily, Keiro, and DuckDuckGo engines with stealth/browser fetch modes
- TaskMaster — Context-isolated pipelines preventing memory bloat in local LLMs
- E*TRADE Integration — Full account, options Greeks, and transaction history access
- Travel Booking — LiteAPI integration for hotel search, rates, and booking flows
- Visualization — Inline SVG, Chart.js, and HTML applets with dark theme support
- Plugin System — Dependency graph management with hot-reload and cycle detection
- Source Tools — Safety-tiered file operations and shell execution with approval gates
- Precision Mode — Vectorized Pandas rules and strict script limits for data tasks
