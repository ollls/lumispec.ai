# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 introduces a production-ready AI assistant featuring a dual LLM backend supporting both local `llama.cpp` inference and cloud `Claude API` routing. The release includes a robust web research system with three search engines (Tavily, Keiro, DuckDuckGo) and three fetch modes (Regular, Stealth, Browser) for comprehensive data gathering. Key integrations span E*TRADE brokerage for full account management and options trading, LiteAPI for travel booking, and a modular plugin architecture with hot-reload capabilities. The system enforces strict precision mode rules for Python execution, mandates vectorized operations, and provides a visualization suite for rendering SVG, Chart.js, and HTML applets.

## Key topics
- Web Research — Three search engines and fetch modes (Stealth/Browser) for bot-resistant scraping
- E*TRADE Integration — Full brokerage access including options Greeks, cost basis, and real-time quotes
- Travel Booking — LiteAPI integration for hotel search, rates, and booking flows with fraud checks
- Visualization — Inline iframe applets supporting SVG, Chart.js, and custom HTML with dark themes
- Dual LLM Backend — Hybrid routing between local `llama.cpp` models and cloud `Claude API`
- Plugin Architecture — Modular tool system with dependency graphs, cycle detection, and hot reload
- Precision Mode — Strict Python guidelines requiring vectorized Pandas and limiting script complexity
- Source Code Tools — Safe file operations with approval tiers for local writes and remote git actions
