# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 introduces a production-ready AI assistant featuring a dual LLM backend supporting both local `llama.cpp` inference and cloud `Claude API` routing. The release highlights the TaskMaster system, which isolates context windows to maintain constant memory footprints for low-resource local models, and a robust Web Research System utilizing Tavily, Keiro, and DuckDuckGo with three distinct fetch modes including Puppeteer-based browser execution. Comprehensive integrations now cover E*TRADE brokerage data with full options Greeks, LiteAPI travel booking with fraud checks, and a modular plugin architecture with hot-reload capabilities. The system also includes a precision-focused Python execution environment, a hierarchical Wiki RAG plugin, and a visualization suite rendering SVG, Chart.js, and custom HTML applets.

## Key topics
- TaskMaster pipelines — Context isolation for low-memory local LLMs
- Web research modes — Stealth, Regular, and full Browser (Puppeteer) fetching
- E*TRADE integration — Real-time quotes, options Greeks, and transaction history
- Travel booking — LiteAPI integration for hotels, rates, and fraud checks
- Dual LLM backend — Hybrid routing between local models and Claude API
- Plugin architecture — Dependency graphs, hot reload, and cycle detection
- Visualization applets — Inline SVG, Chart.js, and HTML rendering
- Precision mode — Vectorized Pandas rules and strict script limits
