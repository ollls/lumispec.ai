# LLM Workbench v1.0.0 Release Notes

LLM Workbench v1.0.0 is a production-ready AI assistant featuring a dual LLM backend (local llama.cpp and cloud Claude) and a modular plugin architecture for extensible tooling. The release introduces a robust `source_*` tool family for self-aware code management, allowing the system to read, edit, and version control its own implementation while enforcing safety tiers for destructive operations. It implements the TaskMaster context-aware pipeline to isolate tasks and maintain constant memory footprints for local LLMs, alongside a sophisticated web research system supporting Tavily, Keiro, and DuckDuckGo with stealth and browser-based fetching modes. Comprehensive integrations include full E*TRADE brokerage access for trading and portfolio analysis, LiteAPI for travel booking, and a hierarchical RAG wiki system that enables a self-improving knowledge loop inspired by Andrej Karpathy's methodology.

## Key topics
- Source tools — Self-modifying code management with safety gates and diff previews
- TaskMaster — Context-isolated pipelines for low-memory local LLM efficiency
- Web research — Multi-engine search with stealth and Puppeteer browser modes
- E*TRADE integration — Full brokerage API for accounts, options, and transactions
- Travel booking — LiteAPI integration for hotel search, rates, and reservations
- Dual backend — Hybrid routing between local inference and cloud Claude API
- Wiki RAG — Hierarchical documentation system with automated scanning and indexing
- Plugin architecture — Hot-reloadable modules with dependency graph management
